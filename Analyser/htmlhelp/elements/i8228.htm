Микросхема КР580ВК28

 

Микросхема КР580ВК28 (аналог Intel 8228) - системный контроллер. Структурная схема изобра­жена на рис. 1. Контроллер состоит из 8-разрядного приемопередатчика (Transceiver), 6-разрядного асинхронного потенциального регистра памяти (Status Larch) и дешифратора DC. Слово состояния SW, выдаваемое МП 8080А по шине данных D7-0 в первом такте каждого ма­шинного цикла (при значении SYNC = 1), записывается в регистр сигналом STSTB = 0. Дешиф­ратор DC, подключенный к регистру, вырабатывает по коду SW и сигналам DBIN и WR активный уровень одного из пяти системных сигналов управления MEMR, MEMW, I/OR, I/OW, INTA и производит переключения направления передачи данных в приемопередатчике.

Ввод данных в МП (рис. 2) указывается значением сигнала DBIN =1 и выполняется одним из сигналов управления: MEMR = 0 (чтение памяти — выборка команд программы и чтение данных), I/OR = 0 (чтение данных из внешних устройств) и INTA = 0 (выборка ко­манд CALL addr или RST n из специального внешнего устройства, называемого контролле­ром прерываний). Электрические и временные параметры всех этих трех сигналов одинаковы, поэтому сигнал MEMR можно использовать и для чтения данных из внешних устройств при отображении их адресов на адресное пространство памяти. Сигнал INTA предназначен для чтения данных из одного единственного внешнего устройства, поэтому эта операция не адресуется по шине адреса МП (безадресное чтение).

Если контакт INTA подключить через резистор 1 кОм к источнику питания +12 В, то на запрос прерывания INT= 1 от внешнего устройства из системного контроллера в МП поступит команда RST 7, машинный код которой равен FFh. Системная шина данных контроллера (System Data Bus) при этом блокируется. Этим обеспечивается одноуровневая система прерываний в простейших микроконтроллерах.

Вывод данных из МП указывается значением сигнала записи WR = 0 и выполняется одним из сигналов управления MEMW = 0 или I/OW = 0. Все параметры этих сигналов одинаковы. Сигналы MEMW и I/OW, выдаваемые системным контроллером 8238, показаны на рис. 2 штриховой линией (упреждающая запись), а контроллером 8228 — сплошной.

 

В   ВК28 (аналог Intel 8228) сигналы чтения памяти MR,чтения портов I/OR и вектора INTA формируется с «упреждением», начало сигнала совпадает с моментом записи байта состояния в регистр, а окончание – с окончанием  сигнала DBIN. Упреждающие сигналы управления позволяют раньше начать выдачу данных из памяти или портов на ШД и стабилизация данных из-за переходных процессов заканчивается раньше. Это уменьшает вероятность  неправильного (т.е. сбоя ) обмена данными с процессором. Другая БИС КР580ВК38 (аналог Intel 8238) отличается от КР580ВК28 формированием упреждающих сигналов управления как чтением так записью и используется в развитых МП системах. Нужно учитывать упреждающее формирование управляющих сигналов записи при использовании некоторых типов памяти, т.к. они выдаются раньше, чем готовы данные для записи, что может привести к некорректной работе.

 

Коды слова состояния SW представлены в таблице 1. Каждый разряд SW содержит сле­дующую информацию о состоянии (в скобках указаны названия разрядов):

 - подтверждение прерывания (INTA);

 - запись данных в память и вывод данных во внешнее устройство (WO — Write/Output);

 - обращение к стеку (STACK - на шине адреса установлено содержимое SP);

 - подтверждение останова (HLTA);

 - вывод данных во внешнее устройство (OUT – на шине адреса установлено зна­чение port второго байта команды OUT port);

 - выполнение первого машинного цикла (М1 —выборка из памяти первого байта очередной команды);

 - ввод данных из внешнего устройства (INP - на шине адреса установлено значе­ние port второго байта команды IN port);

 - чтение памяти (MEMR).

Из таблицы 1 следует, что в качестве системных сигналов управления можно использовать сигналы:

 

Сигналы HLDA и BUSEN (Bus Enable) используются для управления системным контрол­лером при передаче управления системными шинами контроллеру прямого доступа к памяти (DMAC). Сигнал HLDA = 1, подаваемый от МП переводит все системные сигналы управления в неактивное состояние (высокий уровень). Поступающее затем от DMAC значение сигнала AEN = BUSEN = 1 переводит шину данных  и системные сигналы управления в Z-состояние (AEN - Address Enable).

Системные контроллеры 8228/8238 изготовляются по ТТЛШ технологии и обеспечивают токи выходных сигналов:

 при  и  при  для шины данных МП ,

 при  и  при  для остальных сигналов.

 

В одноплатных микроконтроллерах шины данных и адреса не буферируются, так как все БИС обеспечивают на своих выходах токи, достаточные для управления всеми подключенны­ми устройствами ( для выходов БИС, изготовляемых по n-МОП технологии). В многоплатных микроконтроллерах существенно увеличиваются паразитная емкость, подключен­ная к шинам, и статическая нагрузка. Паразитные емкости увеличивают длительности фронтов сигналов, а статическая нагрузка изменяет их уровни, поэтому на каждой плате требуется ус­танавливать шинные формирователи (драйверы) и приемопередатчики, выполняемые по ТТЛ-, МТТЛШ- или ТТЛШ-технологиям. На рис. 3 изображена принципиальная схема централь­ного процессорного устройства с буферированными шинами адреса и данных, выполненная на основе МП 8080А.

Для подачи сигнала системного сброса RESET = 1 при включении питания используется RC-цепь (51 кОм, 10 мкФ). Это обеспечивает старт микроконтроллера с адреса 0 и установку триггера INTE в 0 для запрета прерываний. Сброс может производиться и с пульта управления подачей значения сигнала RESIN = 0. С адреса 0 обычно располагается программа тестирова­ния ОЗУ, ПЗУ и всех подключенных интерфейсных БИС (ИС большой степени интеграции). После этого выполняется программа инициализаций некоторых устройств, приводящая микро­контроллер в рабочее состояние.

Предполагается, что все устройства, подключенные к центральному процессорному уст­ройству, имеют высокое быстродействие, поэтому положили сигнал RDYIN = 1. При использо­вании этого сигнала следует знать, что МП производит анализ его значения во втором такте каждого машинного цикла. При обнаружении значения RDYIN = 0 устанавливается значение сигнала WAIT = 1, и в машинный цикл вводятся холостые такты (такты ожидания) до тех пор, пока не будет обнаружено значение RDYIN = 1.

Сигналы HOLD, HLDA и AEN предназначены для подключения контроллера прямого дос­тупа к памяти 8257 или 8237А фирмы Intel (отечественные БИС 580ВТ57 или 1810ВТ37А). Сигнал HOLD анализируется во втором такте каждого машинного цикла и при обнаружении значения HOLD = 1 при RDYIN = 1 после окончания выполнения текущего машинного цикла выдается значение сигнала HLDA = 1 подтверждения DMA - МП переходит в состояние за­хвата (приостановка работы) с переводом своих шин в Z-состояние. Для перевода в Z-состояние системных, шин используется сигнал AEN = BUSEN = 1, поступающий от контроллера прямого доступа к памяти.
