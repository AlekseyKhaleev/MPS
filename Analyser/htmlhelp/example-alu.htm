5.7. АЛУ с полным управлением (ALU_2_scheme.sch)

 

В рамках лабораторных работ по курсу ОЭВМ автором данного плагина была разработана схема микропрограммного автомата АЛУ.

 

Описание схемы

 

Схема состоит из элемента «АЛУ с открытой структурой» и автомата управления.

 


Схема микропрограммного автомата «АЛУ» (*).

 

Состав автомата управления:

 

1)      память микрокоманд  (элемент «память на 1К 64-х битных слов»);

2)      генератор синхроимпульсов (элемент «Генератор последовательности»);

3)      счётчик адреса микрокоманд (элемент «Счётчик с параллельной загрузкой 8-ми битный»)

4)      схема формирования управляющих сигналов для памяти микрокоманд и сигнала «CLK» («выполнить») для элемента «АЛУ с открытой структурой»;

5)      схема формирования управляющего сигнала переноса разряда для  «АЛУ с открытой структурой»;

6)      программатор для памяти микропрограмм;

7)      индикаторы и логический анализатор;

8)      переключатели «EN», «WR»  и «CLK M»;

9)      блоки переключателей для ввода данных в регистры RGA и RGB элемента «АЛУ с открытой структурой».

 

Память микрокоманд

 

Память микрокоманд (1) хранит микрокоманды, управляющие вычислительными процессами в «АЛУ». При подключении согласно схеме (*) имеем следующие соответствие между управляющими входами элемента «АЛУ с открытой структурой» и разрядами данных памяти микропрограмм.

Генератор синхроимпульсов

 

Генератор синхроимпульсов (2) генерирует синхроимпульс, использующийся при формировании управляющих сигналов памяти («RD» и «CS»), управляющего сигнала АЛУ - «CLK» (выполнить) и синхроимпульса «F1» для счётчика адреса микрокоманд.

Счётчик адреса микрокоманд

 

Счётчик адреса микрокоманд (3) используется для организации адресации памяти микрокоманд при работе всей схемы в режиме выполнения микропрограммы. В качестве синхроимпульса «С+» используется сигнал «F1». Работа счётчика разрешается при сигнале «EN» = 1. Необходимо, чтобы при работе счётчика все разряды блока переключателей адреса были установлены в «0».

 


Расположение счётчика микрокоманд на схеме (*).

 

Схема формирования управляющих сигналов для памяти микрокоманд и сигнала «CLK» («выполнить») для элемента «АЛУ с открытой структурой»

 

Используется для соблюдения временной диаграммы (**) и разделения двух режимов работы схемы — режима «микропрограммирование» и режима «выполнение микрокоманд».

 


Схема формирования управляющих сигналов для памяти микрокоманд и сигнала «CLK» («выполнить») для элемента «АЛУ с открытой структурой»  - увеличенный фрагмент (4) схемы (*).

 

Сигналы схемы (4):

 

Входные сигналы

 

1)      F – основной синхросигнал от генератора синхроимпульсов («Clock»);

2)      EN – сигнал от переключателя активации режима «выполнение микропрограммы»;

3)      СLK M – сигнал от переключателя CLK M («выполнить микрокоманду вручную»);

 

Выходные сигналы

 

4)      F1 – синхросигнал С+ счётчика адреса памяти микрокоманд и управляющий сигнал RD памяти микрокоманд;

5)      СLK – сигнал «выполнить» для элемента «АЛУ с открытой структурой»;

6)      CS – сигнал «chip select» для памяти микрокоманд;

 

Вспомогательные сигналы

 

7)      EN1, EN2, F2 – вспомогательные сигналы — результаты промежеточных преобразований при формировании сигналов выходной группы [4-6].

Схема формирования управляющего сигнала переноса разряда для  «АЛУ с открытой структурой»

 

Здесь представлен один из вариантов использования разрядов D0, D7 (последовательные входы SM) и D13, D14, D15 (последовательные выходы SM). Схема реализует перенос из одного из разрядов  D13, D14, D15 в один из разрядов  D0, D7. Такого рода действие может понадобиться для реализации циклического сдвига или для перевода операнда из прямого кода в дополнительный код.

6)      С2->C4 –  сигнал из памяти микрокоманд «прибавить к старшей части данных SM  разряд D14 (sig SM)»;

Программатор для памяти микропрограмм

 

Программатор для памяти микропрограмм (6) это дополнительные блоки переключателей ручного набора микрокоманд и адреса памяти микропрограмм, которые используются для упрощения создания процедуры создания микропрограммы.

 


Расположение программатора на схеме (*).

 

Алгоритм использования программатора:

 

1)      Для корректного использования  программатора необходимо, чтобы переключатель EN был установлен в «0», а переключатель WR в «1».

2)      На трёх блоках переключателей, соответствующих управляющим сигналам RGA, RGB и SM элемента «АЛУ с открытой структурой» набираем микрокоманду.

3)      На блоке переключателей адреса выставляем адрес памяти микропрограмм, на который мы хотим записать эту микрокоманду.

4)      Переводим переключатель WR в «0».

5)      Возвращаем переключатель WR в «1».

 

После выполнения вышеперечисленных действий набранная микрокоманда запишется на указанный адрес.

 

Систему переключателей программатора можно использовать для ручного выполнения микрокоманды (заданной на его переключателях). Для этого необходимо выполнить следующую последовательность действий:

 

1)      Переключатели EN  и CLK M должны быть установлены в «0», а переключатель WR в «1».

2)      На трёх блоках переключателей, соответствующих управляющим сигналам RGA, RGB и SM элемента «АЛУ с открытой структурой» набираем микрокоманду.

3)      Устанавливаем  CLK M в «1».

 

После выполнения вышеперечисленных действий набранная микрокоманда выполнится схемой «АЛУ».

 

Индикаторы и логический анализатор

 

Используются для мониторинга схемы. На цифровые индикаторы (2 шт.) выводится адрес микрокоманды, а логический анализатор используется для наблюдения временных диаграмм сигналов F1, CLK, CS.

 

Переключатели «EN», «WR»  и «CLK M»

 

Переключатели EN и WR используются для переключения режимов схемы:

 

1)      EN=1, WR=1 – режим «выполнение микропрограммы»;

2)      EN=0, –  режим «микропрограммирование» | «ручное выполнение».

 

Переключатель CLK M используется для выработки сигнала CLK при ручном выполнении программ.

 

Для запуска выполнения микропрограммы необходимо выполнить следующую последовательность действий:

 

1)      выключить режим симуляции (если он был включен);

2)      установить переключатели EN=0, WR=1;

3)      запустить симуляцию;

4)      установить переключатель EN=1.

 

Использование этих переключателей подробно описано в пункте «программатор для памяти микропрограмм».

 

Блоки переключателей для ввода данных в регистры RGA и RGB элемента «АЛУ с открытой структурой»

 

Два блока переключателей, использующиеся для набора данных, которые впоследствии вводятся в регистры RGA и RGB элемента «АЛУ с открытой структурой». Эти блоки переключателей доступны для использования при любом режиме работы схемы.

 

Разработка микропрограммы

 

Рекомендуемый алгоритм разработки микропрограммы

 

1)      определить из каких микрокоманд должна состоять микропрограмма;

2)      определить порядок микрокоманд в микропрограмме;

3)      выключить симуляцию;

4)      установить режим «микропрограммирование» | «ручное выполнение»;

5)      включить симуляцию;

6)      проверить каждую составленную микрокоманду в режиме ручного выполнения;

7)      соблюдая порядок микрокоманд, записать их в память с помощью программатора

 

Пример разработки микропрограммы

Пример 2. Необходимо разработать микропрограмму, реализующую операцию умножения.

 

В данном примере будут рассмотрены только основные принципы потроения такой микропрограммы

 

Умножение начиная с младших разрядов со сдвигом содержимого сумматора направо.

 

1)      очистка сумматора, приём операндов во входные регистры;

2)      i=0...7 умножаем содержимое одного регистра на i-й разряд другого регистра, складываем результат со старшей частью сумматора, сдвигаем содержимое сумматора направо на 1 разряд;

3)      определяем знак результата, реализуя функцию {f(0,0)=f(1,1)=0; f(1,0)=f(0,1)=1};

4)      записываем старшую часть результата в RGB, младшую в RGA (вместе со знаком результата).

 

Умножение начиная со старших разрядов со сдвигом содержимого сумматора влево.

 

1)      очистка сумматора, приём операндов во входные регистры;

2)      i=7...0 умножаем содержимое одного регистра на i-й разряд другого регистра, складываем результат с содержимым сумматора, сдвигаем содержимое сумматора влево на 1 разряд (на последней итерации сдвиг не нужен);

3)      определяем знак, реализуя функцию {f(0,0)=f(1,1)=0; f(1,0)=f(0,1)=1};

4)      Записываем старшую часть результата в RGB, младшую в RGA (вместе со знаком результата)

 

Подсказка при x={0 | 1} умножение семиразрядного бинарного числа abcdefg на x эквивалентно операции  abcdefg & xxxxxxx (где & - операция побитового «и»).
